Introduction Last updated: 2019-06-01

The purpose of this class is to create an easy to use method for utilizing Microsoft's WebView2 Framework in AHK v2, allowing you to create web-based GUIs and applications without relying on ActiveX.

This class is built off of Thqby's WebView2 class, which does the heavy lifting of making the WebView2 Framework available in AHK. Their class however has a higher skill-floor than I found desirable, so I went on to write this class, off of theirs, to make it easier to use by people of all skill levels.

Installation

There are a few resources you'll need to download and setup in order to use this class. You can download the files directly from GitHub at the following locations:

  • WebView2 Class by Thqby
  • ComVar Class by Thqby
  • WebViewToo Class by The-CoDingman
You can also download a ZIP folder I've put together with everything ready to go here: ZIP'd Package
After you'd gotten everything downloaded, you are ready to move on to making your first WebView2 GUI!

Load

Navigates to a webpage or file.
MyWindow.Load(Url , DeleteFiles)

Parameters

    Url
  • Type: String
  • Url of webpage or path to file you want to navigate to
    DeleteFiles
  • Type: Integer (boolean)
  • If blank or omitted, this defaults to 0. When set to 1 and you're using a complied .exe with files that have been included using FileInstall the files will be deleted after a short delay.

simplePrintToPdf

Triggers a simplified PrintToPdf event for saving the webpage as a PDF file.
MyWindow.simplePrintToPdf(Filename, Orientation)

Parameters

    Filename
  • Type: String
  • If omitted, the end-user will be prompted via Fileselect() to choose where they want to save the file and what they want to name it. Otherwise, specify the filepath, including filename and extension, that you want to save the printed webpage to.
    Orientation
  • Type: String
  • If omitted, the page orientation will default to Portrait. Otherwise, specify the page orientation you would like to save the pdf as, either "Landscape" or "Portrait".

Remarks

    If you need to perform a more in-depth or specialized PrintToPdf event, use the PrintToPdf() method discussed in the [later section].

CreateTempHtmlFile

Used to create a temporary HTML file which can be used to navigate to.
WebviewWindow.CreateTempHtmlFile(Filename, Contents , Timeout)

Parameters

    Filename
  • Type: String
  • If omitted, this will be set to the WebviewWindow.TEMPLATE.NAME. Otherwise, specify the full filepath, inlcuding name and extension, that you wish to create.
    Contents
  • Type: String
  • If omitted, this will be set to the WebviewWindow.TEMPLATE.CODE. Otherwise, specify the entire contents of the HTML file you want to create.
    Timeout
  • Type: Integer
  • If omitted, this will be set to 10000 milliseconds. Otherwise, specify in milliseconds how long to wait before deleting the created HTML file.

Remarks

    Because of the permission settings inherit with WebView2's NavigateToString() method, there may be times you find it more useful to navigate to a local file. This method allows you to create an HTML file from string, and then delete it after it has been loaded.

Parameters

    WebviewWindowObject
  • Type: Object
  • The WebviewWindow that you want to get the parent of.

Remarks

Formats provided strings and converts certain characters into HTML entities.

Parameters

SetBoundsAndZoomFactor

Updates Bounds and ZoomFactor properties at the same time
MyWindow.SetBoundsAndZoomFactor(Bounds, ZoomFactor)

Parameters

    Bounds
  • Type: Rectangle
  • The bounds to be updated
    ZoomFactor
  • Type: Double
  • The zoom factor to be updated

Remarks

    This operation is atomic from the perspective of the host. After returning from this function, the Bounds and ZoomFactor properties are both updated if the function is successful, or neither is updated if the function fails. If Bounds and ZoomFactor are both updated by the same scale (for example, Bounds and ZoomFactor are both doubled), then the page does not display a change in window.innerWidth or window.innerHeight and the WebView renders the content at the new size and zoom without intermediate renderings. This function also updates just one of ZoomFactor or Bounds by passing in the new value for one and the current value for the other.

    You can view the MSDN documentation for this method here.

MoveFocus

Moves focus into WebView.
MyWindow.MoveFocus(Reason)

Parameters

    Reason
  • Type: CoreWebView2MoveFocusReason(Integer)
  • The reason for moving focus.
  • Specify one of the following values:
  • 1: Specifies that the focus is moved due to Tab traversal forward.
  • 2: Specifies that the focus is moved due to Tab traversal backward.
  • 3: Specifies that the code is setting focus into WebView.

Remarks

    WebView will get focus and focus will be set to correspondent element in the page hosted in the WebView. For Programmatic reason, focus is set to previously focused element or the default element if no previously focused element exists. For Next reason, focus is set to the first element. For Previous reason, focus is set to the last element. WebView changes focus through user interaction including selecting into a WebView or Tab into it. For tabbing, the app runs MoveFocus with Next or Previous to align with Tab and Shift+Tab respectively when it decides the WebView is the next tabbable element.

    You can view the MSDN documentation for this method here.
